1.1 · Place Value

🧪 PRETEST

Place value: digit position determines value. Base-10 uses powers of 10.

3,524 = 3×10³ + 5×10² + 2×10¹ + 4×10⁰ = 3000 + 500 + 20 + 4

Pattern: Each position left is 10× larger. Ones → Tens → Hundreds → Thousands

Positional Notation Systems

A positional numeral system represents numbers where digit position determines magnitude. Modern decimal (base-10) employs ten symbols (0-9), each position representing power of base.

Digit position determines magnitude through base exponentiation. In base-10 (decimal), each position represents 10ⁿ where n is the position index: 10⁰=1, 10¹=10, 10²=100, 10³=1000. The digit "3" carries different weight depending on its position: 3×10⁰=3, 3×10³=3000.

The key insight: moving one position left multiplies value by the base. In decimal, that's ×10 each step. In binary (base-2), it's ×2 each step. This positional principle generalizes to any base: binary uses positions 2⁰, 2¹, 2², 2³ (evaluating to 1, 2, 4, 8), while decimal uses powers of 10.

Compare: decimal 13 = 1×10¹ + 3×10⁰, while binary 1101 = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ = 13 (same value, different representation). Alternative bases appear throughout mathematics: binary in computer science and boolean algebra, hexadecimal in memory addressing, and base-60 in ancient Babylonian astronomy. You'll encounter binary extensively in modular arithmetic (Path 300) and computational theory.

Formal: N = Σ(dᵢ × bⁱ) for i=0 to n

Where dᵢ = digit at position i, b = base

Breaking down the formula:

  • N = the number we're representing
  • dᵢ = digit at position i (ranges 0-9 in decimal)
  • b = base of the system (10 for decimal)
  • i = position index (starts at 0 from right)
  • Σ = sum all products together

Example decomposition (3,524):

  • d₃=3, position 3: 3×10³ = 3000
  • d₂=5, position 2: 5×10² = 500
  • d₁=2, position 1: 2×10¹ = 20
  • d₀=4, position 0: 4×10⁰ = 4

Alternative bases: binary (base-2), octal (base-8), hexadecimal (base-16). Historical: Babylonian (base-60), Mayan (base-20).

Place value is like organizing your piggy bank. Where you put the coins matters!

📍 Example 1: Sorting 2,743 Pennies

Okay, I've got 2,743 pennies spread all over my floor. Let me organize them!

First, I make piles of 1,000:
1 pile... 2 piles! ✅ Now I have 743 left over.

Next, piles of 100:
1... 2... 3... 4... 5... 6... 7 piles! ✅ Now I have 43 left over.

Then piles of 10:
1... 2... 3... 4 piles! ✅ Now I have 3 left over.

What's left?
Just 3 single pennies sitting there.

2,743 = 2 thousands + 7 hundreds + 4 tens + 3 ones

🎯 The Pattern:

Each place is 10 times bigger than the one to its right. You fill up the small piles first, then when you have 10 of something, you make 1 pile of the next size!

1 → 10 → 100 → 1,000 → 10,000 (keeps going forever!)

📍 Example 2: Street Address

Think of it like a street address: 5,204 Main Street

• 5 = what thousand block you're on (5,000s)

• 2 = what hundred within that block (5,200s)

• 0 = what ten (no tens - you're at the start!)

• 4 = exact house number (the 4th house)

Same digits, different spots = you're at a completely different house!

(5,204 Main St is NOT the same as 4,025 Main St)

⚡ Why This Matters:

If you mix up the positions, 52 becomes 25. That's like showing up to house #52 when you meant house #25 - you're at the wrong place!

In math, getting the position wrong means getting the WHOLE answer wrong, even if you have the right digits.

💡 Pro Reading Trick:

Start from the RIGHT and count: ones, tens, hundreds, thousands.

For 8,512: Start at 2 (ones) → 1 (tens) → 5 (hundreds) → 8 (thousands)

Once you get the rhythm, you can read ANY number!
